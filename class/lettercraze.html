LetterCraze

Overview

Description: A puzzle game similar to Boggle where the player must form words in a grid of letters. After a word is formed, the letters disappear, and those on top of them shift down. There are two elements to this project: a player (the game itself) and a level builder, which we used to make 15 levels. Additionally, the game features three different modes, each of which feature a high score stored in a table. The score would determine a star level, from 0-3.:

  • Theme: The letters on the board form an exact set of words, which are shown to the player. They must try to form all those words with the letters given.
  • Lightning: There is a time limit, after which the game is over. The player tries to beat a high score.
  • Puzzle: There is limited number of moves available, after which the game is over. The player tries to beat a high score.
The builder was responsible for setting several properties in a level:
  • The size of the board
  • Which squares are valid (some are blocked and always empty)
  • The star levels for lightning and puzzle modes
  • The words and placement of letters for theme mode

Assignment Requirements: The requirements for the features were clearly given, and we followed those closely. The challenge was designing the application, as we were given full autonomy here. We began by coming up with all the use cases for both the game and the builder, and then created detailed UML diagrams to show all of the classes involved. We then implemented those classes, with each team member working on a different component (model, view, and controllers).
