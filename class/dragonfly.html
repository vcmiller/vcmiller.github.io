Dragonfly Engine

Overview

Description: A 2.5-D game engine. While it is simple, its capabilities are sufficient that a team member and I were able to use it to create our final project for the class. The capabilities of the engine include:

  • Loading and rendering ASCII files as sprites.
  • Sprite frame animation.
  • Rendering in parallax mode (object size scales depth and position).
  • Collision detection with oriented bounding boxes.

Links:

  • Source: I am unable to provide the source for the engine, in accordance with WPI's policy on sharing source code for projects.
  • Game Executable: Direct download

Team: The engine itself was a solo project. For the final class project, a game that used the engine, I worked with one partner (Sean Briggs).

Technologies Used:

  • C++ (Windows)
  • SFML

Class: Technical Game Development 1 (IMGD 3000)

Assignment Requirements: The requirements of this assignment, as well as high-level design, were clearly specified. However, I made several additions of my own:

  • 2.5-D rendering using a parallax projection.
  • Support for rotation and scaling of objects, including collision detection.
  • Joystick and gamepad input support.
